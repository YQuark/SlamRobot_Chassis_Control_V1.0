# AGENTS.md

## 项目定位

本项目为两轮差速移动机器人底盘控制工程（`YQuark/SlamRobot_Chassis_Control_V1.0`），基于原有 `YQuark/SLAM_ROBOT` 工程（四轮差速，2026-04 归档只读）继续开发，但当前硬件架构已经发生较大变更，不应直接沿用旧工程中的四轮底盘、电机驱动、编码器参数与控制假设。

关键变更：四轮差速 → 两轮差速；裸机/简单调度器 → FreeRTOS CMSIS-RTOS v2；COBS+CRC16 协议 → 0xA5 0x5A + CRC8-MAXIM 二进制帧；驱动芯片 → TB67H450FNG；电机 → JGB37-520。

当前工程已经通过 STM32CubeMX 生成初始代码，并在此基础上继续进行驱动层、控制层、通信层、FreeRTOS 任务层与上位机接口适配。

本项目目标是实现一套面向 ROS2 上位机控制的两轮差速底盘控制系统，支持本地 PS2 控制、ESP01S 通信、树莓派上位机串口控制、电机闭环控制、电压/电流采样、IMU 姿态数据读取与基础状态反馈。

## 当前硬件架构

### 1. 主控与系统框架

- 主控 MCU：STM32F407VET6 (Cortex-M4F, 168MHz, 512KB Flash, 128KB SRAM + 64KB CCMRAM, LQFP100)，当前工程已由 STM32CubeMX 生成初始化代码。

- 软件框架：HAL + FreeRTOS。

- 项目基底：参考 `https://github.com/YQuark/SLAM_ROBOT`，但当前版本已经更换为两轮差速结构。

- 控制目标：左右双电机独立控制，构成两轮差速底盘。

- 上位机：树莓派 4B。

- 上位机系统：Ubuntu 22.04 + ROS2。

- 上位机通信方式：树莓派 GPIO 串口连接 MCU 的 USART3。

---

### 2. 底盘结构

当前底盘为两轮差速结构：

- 左轮：独立电机驱动，带 AB 相编码器反馈。

- 右轮：独立电机驱动，带 AB 相编码器反馈。

- 非四轮差速。

- 非麦轮结构。

- 不应保留旧工程中四轮底盘、四路电机、四路编码器相关逻辑。

- 任何速度解算、里程计解算、PID 控制、串口协议、状态反馈都应按“两轮差速”重新适配。

### 4. 编码器参数

当前电机自带 AB 双相增量式霍尔编码器。

编码器参数：

- 类型：AB 双相增量式磁性霍尔编码器。

- 输出信号：方波 AB 相。

- 编码器供电：DC 3.3 V 或 DC 5.0 V，具体以当前原理图供电为准。

- 基础脉冲数：11 PPR。

- 磁环触发波数：22，即 11 对极。

- 编码器输出频率能力：约 100 kHz。

注意：

- 电机减速比为 56。

- 电机轴端编码器基础分辨率为 11 PPR。

- 当前使用 TIM 编码器模式（AB 相 4x 正交），输出轴每圈计数约为：

```text
11 × 4 × 56 = 2464 counts/rev
```

- 若修改编码器模式或倍频设置，必须同步更新 `CHASSIS_ENCODER_QUADRATURE_MULT` 和换算系数。

- 不得在未确认计数模式前固定写死编码器换算系数。

- 需要在代码中明确区分：
        

    - 编码器原始 PPR。

    - AB 相倍频倍率。

    - 减速比。

    - 输出轴每圈计数。

    - 左右编码器方向符号。

### 5. 电机驱动芯片

当前电机驱动芯片：

- 型号：TB67H450FNG,EL。

- 每个电机使用一路 H 桥驱动。

- 当前为双电机，因此需要两路 TB67H450FNG 或等效双路驱动电路。

- 每个电机由两路 PWM 输入控制（双路 PWM 模式，IN1/IN2 各自独立 PWM，非 PWM+DIR 模式）。

- OUT1 / OUT2 接电机两端，实际转向可通过软件方向符号修正。

- 不应假设 OUT1/OUT2 与左右方向天然一致，必须提供左右电机方向校准参数。

控制原则：

- 两个输入信号控制一个电机的正转、反转、刹车或滑行状态。

- PWM 控制应支持：


    - 左电机正转。

    - 左电机反转。

    - 右电机正转。

    - 右电机反转。

    - 停止。

    - 制动或空转策略，按当前驱动逻辑统一实现。

- 代码中应封装电机驱动层，不允许在上层控制逻辑中直接散落 GPIO/PWM 操作。

### 10. 控制输入

当前保留三类控制来源：

1. PS2 手柄控制。

2. ESP01S 通信控制。

3. 树莓派 ROS2 上位机串口控制。

要求：

- 需要设计统一控制模式管理。

- 不同控制源不应同时直接写电机输出。

- 应建立统一的目标速度结构体，例如：

```c

{

    float angular_z;
    uint8_t source;
    uint32_t timestamp_ms;

```

- 底层只接受统一后的底盘速度目标或左右轮速度目标。

- 建议控制优先级后续明确，例如：
        

    - 急停最高。

    - 上位机控制。

    - PS2 手动控制。

    - ESP01S 控制。

- 在未明确优先级前，不要写死复杂抢占逻辑。

### 11. FreeRTOS 架构要求

当前项目引入 FreeRTOS。

建议任务划分：

```text
Task_ChassisControl
- 周期执行底盘速度闭环。
- 读取目标速度。
- 计算左右轮目标。
- 执行 PID。
- 输出 PWM。

Task_EncoderUpdate
- 周期读取或处理编码器计数。
- 计算左右轮速度。
- 更新里程计基础数据。

Task_AdcMonitor
- 周期采样电池电压、左电机电流、右电机电流。
- 更新系统电源状态。
- 执行低压、过流、堵转判断。

Task_ImuUpdate
- 读取 MPU6050 数据。
- 执行基础滤波或零偏修正。

Task_UpperUart
- 处理 USART3 上位机收发。
- 解析控制命令。
- 上传底盘状态。

Task_Esp01s
- 处理 USART2 通信。

Task_Ps2
- 读取 PS2 控制输入。

Task_Led
- 呼吸灯状态显示。
```

要求：

- 不得在多个任务中直接同时操作电机 PWM。

- 电机输出只能由底盘控制任务统一执行。

- 通信任务只更新目标命令，不直接控制电机。

- ADC、IMU、编码器等传感器任务只更新状态数据。

- 共享数据必须考虑并发访问，可使用互斥锁、临界区、消息队列或双缓冲。

- 不应在中断中执行复杂计算。

- 串口接收建议使用中断或 DMA，再由任务解析协议。

- PID 控制周期应稳定，不应被日志输出、串口阻塞或传感器读取阻塞。

**数据流（USART3 控制链路）：**

```text
USART3 DMA RX ISR
  → UpperUart 任务解析帧
    → ControlManager 更新 chassis_cmd（临界区保护）
      → ChassisControl 任务读取目标速度
        → 差速解算 v_left / v_right
          → motor_driver SetPermille
            → TIM PWM 输出
```

并发保护约定：ControlManager 使用临界区保护 `chassis_cmd` 读写；传感器状态结构体（`adc_monitor_state_t`、`encoder_state_t`、`imu_mpu6050_state_t`、`system_monitor_state_t`）使用 `__get_PRIMASK`/`__disable_irq`/`__set_PRIMASK` 临界区保护整体写入和整体拷贝，防止多任务并发读取时发生撕裂；电机 PWM 只由 ChassisControl 任务操作。

### 13. 两轮差速运动学

当前底盘为标准两轮差速。

应使用以下抽象：

```text
v  = 线速度
w  = 角速度
L  = 左右轮距
r  = 轮半径

v_left  = v - w * L / 2
v_right = v + w * L / 2
```

注意：

- 轮距 L、轮半径 r 必须作为配置项。

- 当前若机械结构尚未完全定型，不应写死最终参数。

- 编码器速度换算应依赖实际轮径、减速比和计数模式。

- 左右电机方向可能不同，必须提供方向符号修正。

- 若发现 ROS2 中前进、后退、左转、右转方向错误，应优先调整方向配置，而不是改动高层运动学公式。

### 15. PID 与闭环控制要求

当前项目需要支持编码器闭环控制。注意：当前底盘控制为开环（PWM 直接比例映射），PID 闭环控制为第二阶段目标，在编码器速度计算未验证前不应实施。

要求：

- 左右电机分别独立 PID。

- PID 输入：左右轮实际速度。

- PID 目标：左右轮目标速度。

- PID 输出：左右电机 PWM 占空比或等效控制量。

- 支持输出限幅。

- 支持死区补偿。

- 支持方向切换。

- 支持急停清零。

- 支持编码器方向反向配置。

- 支持电机方向反向配置。

不应在未验证前直接套用旧工程 PID 参数。

初始调试时应支持：

- 单电机开环测试。

- 单电机编码器方向测试。

- 单电机速度闭环测试。

- 左右双电机同向前进测试。

- 原地左转/右转测试。

- 上位机速度指令测试。

### 17. 上位机 ROS2 适配方向

树莓派 4B 运行 Ubuntu 22.04 + ROS2。

MCU 侧应提供稳定串口协议，服务于 ROS2 底盘节点。

建议上位机发送：

```text
linear_x
angular_z
enable
mode
checksum
```

建议 MCU 返回：

```text
left_speed
right_speed
left_encoder
right_encoder
battery_voltage
left_current
right_current
imu_data
error_flags
control_mode
```
