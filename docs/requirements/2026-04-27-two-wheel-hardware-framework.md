# 两轮差速底盘硬件框架需求

## 目标

在现有 STM32F407 CubeMX、HAL、FreeRTOS 工程上新增两轮差速底盘首轮硬件框架，为后续电机闭环、ROS2 上位机协议、PS2 和 ESP01S 接入提供稳定代码路径。

## 交付内容

- 新增 BSP/App 分层目录和模块骨架。
- 集中配置两轮差速、电机、编码器、ADC 和保护阈值参数。
- 封装 TB67H450FNG 双输入 PWM、TIM 编码器、ADC DMA、LED 状态接口。
- 新增底盘控制、控制源管理、系统监控和上位机协议的应用层接口。
- 在 FreeRTOS 中接入底盘控制、编码器更新、ADC 监控和 LED 任务。
- 保持 CubeMX 生成文件只在 USER CODE 区做最小接入。

## 约束

- 底盘结构固定为左右两轮差速，不引入四轮、麦轮或 SLAM 算法。
- 未确认的轮距、轮径、分压比、电流采样比例等参数只能配置化或标记 TODO。
- 通信、PS2、ESP01S、IMU 任务不得直接写 PWM。
- 电机输出只能通过底盘控制层调用电机驱动层。
- 首轮默认安全停车，不在未调试闭环前自动根据速度命令输出 PWM。

## 验收标准

- CMake 能发现并编译新增模块。
- `Core/Src/freertos.c` 的任务接入位于 USER CODE 区。
- PWM、编码器、ADC、LED 都有独立 BSP 接口。
- 上位机协议结构预留左右轮速度、编码器、电压、电流、IMU 摘要、错误状态和控制模式字段。
- 代码中没有旧四轮核心接口和硬编码未确认硬件换算。
